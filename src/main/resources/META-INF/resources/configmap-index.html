Configmap Mission - Red Hat Fuse

This mission provides a basic example of using a ConfigMap to externalize configuration used in an Apache Camel route. ConfigMap is an object used to inject configuration data as simple key and value pairs into one or more Linux containers.

This page is already invoking the Greetings Service every 5 seconds; and you can see the answer "greeting the default value". If you modify the booster.nameToGreet value in the fuse-configmap-booster Configmap using the Kubernetes/Openshift console or CLI tools, you will see the change reflected in the greetings service's answer.
